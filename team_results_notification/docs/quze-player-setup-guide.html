Quze — Player Setup Guide

How to open the game and connect to the server

Two URLs — you need both

Quze uses two separate addresses. Players often need both on first launch.

Purpose What it is
App URL Flutter web PWA — open the game in the browser
API URL FastAPI backend — set once in Database Info on login

1. App URL (open the game in the browser)

This is the Flutter web PWA, not the API.

Local network (your PC hosts the game)

After building and serving build/web:

Setup URL to give players
HTTP (e.g. python -m http.server 8080 --bind 0.0.0.0) http://YOUR_PC_IP:8080/
HTTPS (recommended on phones) https://YOUR_PC_IP:8080/

Replace YOUR_PC_IP with the PC’s LAN address from ipconfig (e.g. 192.168.2.10).

Important: Do not send localhost or 127.0.0.1 — that only works on the PC itself.

Install on phone: open that URL → Add to Home screen / Install app (not browser “Download”).

Production / cloud

Use your hosted PWA URL, for example:

https://quze.yourdomain.com/

(Firebase, Netlify, Nginx, etc. — see CLOUD_DEPLOYMENT.md in the project.)

2. API URL (Database Info on login)

The app talks to FastAPI separately. Default is http://localhost:8000/api, which does not work on a phone.

On the login screen → Database Info, set:

Setup API URL
Local LAN http://YOUR_PC_IP:8000/api
HTTPS backend https://YOUR_PC_IP:8000/api
Production (same domain) https://yourdomain.com/api

Backend must run on 0.0.0.0:8000 and port 8000 must be reachable (firewall).

What to send users (typical LAN setup)

Message example:

Open: http://192.168.2.10:8080/  (or https://... if using HTTPS)

On first launch → Database Info → Server URL:
http://192.168.2.10:8000/api → Save

Log in and play.

Optional: QR code for the app URL (:8080).

Single URL (production, optional)

If Nginx/Caddy serves the PWA at / and proxies /api to FastAPI on the same domain, you can send only:

https://yourdomain.com/

API may still need to be set in Database Info unless the app is configured to use the same origin automatically (default is still localhost:8000 until the user saves a custom URL).

Summary
  • Send players the :8080 app link.
  • They must set the :8000/api server in Database Info on first use.
  • Exception: both deployed under one HTTPS domain with proper config.

Host setup (organizer)

  1. Backend: python backend_fastapi/start_server.py (port 8000)
  2. Web app: serve build/web on port 8080: 
    cd team_results_notification/build/web
python -m http.server 8080 --bind 0.0.0.0
  3. Share http://YOUR_PC_IP:8080/ and http://YOUR_PC_IP:8000/api
