Quze — Debug Mode (Web)

On-screen debug console for troubleshooting

How debug is enabled

Debug mode on the web app is controlled by a URL query parameter, not a separate build flag.

In main.dart, debug is enabled when:

kIsWeb && Uri.base.queryParameters['debug']?.toLowerCase() == 'true'

That turns on an on-screen Debug console overlay (logs, errors, print() output).

URL to send / open

Same app URL as before, with ?debug=true:

Setup URL
HTTP (LAN) http://YOUR_PC_IP:8080/?debug=true
HTTPS (LAN) https://YOUR_PC_IP:8080/?debug=true
Production https://yourdomain.com/?debug=true

Examples:

  • http://192.168.2.10:8080/?debug=true
  • http://192.168.2.10:8080/login?debug=true (also works if you land on login)
Database Info API URL is unchanged — still e.g. http://YOUR_PC_IP:8000/api on first use.

How to run it locally

Step Command / action
Backend python backend_fastapi/start_server.py (port 8000)
Web app Serve build/web on port 8080, or flutter run -d chrome --web-port=8080
Open http://YOUR_PC_IP:8080/?debug=true (PC or phone)

During development

cd team_results_notification
flutter run -d chrome --web-port=8080

Then in the browser add ?debug=true to the address bar (e.g. http://localhost:8080/?debug=true).

Notes

Topic Detail
Web only debug=true has no effect on Android/iOS native builds.
Installed PWA Home-screen icon usually opens without ?debug=true (manifest start_url is .). For debug, open the browser link with ?debug=true each time.
Not Flutter “Debug” banner debugShowCheckedModeBanner is off; you get the custom overlay in the bottom-right instead.
Normal players Use release build without ?debug=true.
Troubleshooting Use ?debug=true only when diagnosing issues.
